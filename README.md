## 🚀 MeterFlow: Technical ArchitectureMeterFlow
provides real-time visibility into API consumption, latency, and billing metrics. 
As seen in image_a229dd.png, the dashboard transforms complex request logs into actionable financial and performance data.
### 1. High-Performance RuntimeThe project utilizes Bun as its primary runtime and package manager.
This ensures near-instantaneous cold starts and high-speed dependency resolution, which is critical for edge-deployed applications.  
### 2. The Full-Stack Meta-FrameworkThe app is built on Vinxi, a versatile SDK that orchestrates Vite and Nitro.  
Nitro: Powers the server-side logic, allowing the app to run seamlessly on Cloudflare Workers.  
Vite: Handles the frontend bundling for the React-based dashboard. 
### 3. Edge-First DeploymentDeployment is optimized for Cloudflare.
The inclusion of .wrangler/ and .dev.vars in the configuration confirms a serverless architecture where code executes at the network edge to minimize global latency. 
### 4. Type-Safe Routing & StateTanStack Router: Provides 100% type-safe routing, managing complex dashboard states and data pre-fetching.  
Redux + Immer: Manages global application state (like user sessions and billing data) with immutable update patterns. 
### 5. Advanced UI ComponentsThe interface (visible in image_a229dd.png) is built using Radix UI primitives for accessibility and Tailwind CSS for styling. Key components include:  vaul: Responsive drawers for mobile views.  cmdk: An "Apple-style" command palette (Cmd+K) for rapid navigation.  sonner: High-performance toast notifications.  ### 6. Data VisualizationTo render the request graphs seen in the dashboard, the project integrates D3.js and Recharts. This allows for granular control over time-series data visualization.  ### 7. Schema ValidationZod is used throughout the project to validate incoming API request shapes and environment variables, ensuring that the "Request" and "Error Rate" metrics are always accurate.  ### 8. ### 10. Utility IntegrationThe logic layer is bolstered by specialized utilities:  date-fns: For calculating "Requests · 30d" time ranges.  nanoid: For generating unique API keys or request IDs.
